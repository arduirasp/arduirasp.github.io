Mise en place d'un serveur git

Published: ven. 01 janvier 2016
By Dominique

In Raspberry.

tags: installation git

Informations

Nous allons aller à l'essentiel. Il faut supposer que le Rpi est relié au réseau par une adresse IP fixe.

Le mieux serait de créer un utilisateur dédié. Nous allons juste mettre en place les dépôts vides ainsi que les clé publiques ssh.

Sur le Raspberry

Créer le dépôt

Tout d'abord, il nous faut créer le répertoire où mettre le projet : commande mkdir ; puis aller dans le répertoire créé avec la commande cd. Et enfin initialiser le dépôt avec la commande git --bare init. Attention, la commande contient aussi --bare qui crée un dépôt sans répertoire de travail contrairement au dépôt sur nos PC.

Autoriser les utilisateurs à se connecter au serveur

Pour cela, nous allons le fichier authorized_keys dans le dossier .ssh. Habituellement, ce dossier n'est pas créé. Nous allons donc créer le dossier avec mkdir .ssh et le fichier avec nano nomdufichier.

Dans ce fichier, la clé publique ssh de chaque utilisateur sera copié.
Sur le PC de chaque utilisateur, la clé publique se trouve dans le fichier id_rsa.pub dans le dossier .ssh. Copier le contenu de ce fichier dans le fichier authorized_keys sur le Rpi.

Sur le PC de l'utilisateur

Dans le dossier du dépôt git, il nous faut ajouter ce nouveau dépôt distant sur notre serveur perso avec la commande : git remote add perso pi@adresseip:/home/pi/git/depotprojet

Puis avec la commande git push perso master ou autre branche pour pousser les modifications et git pull perso master pour récupérer les modifications.

Notes

Il nous faudra revenir plus en détail plus tard sur une installation propre de Raspbian pour permettre une sécurité accrue.
