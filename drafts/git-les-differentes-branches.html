Git : les différentes branches

Published: mer. 17 février 2016
By Dominique

In GNU/Linux.

tags: git

Quelle utilité ?

Lors de l'écriture d'un document en l'occurence d'un code, il arrive à un moment où le code compile correctement et que nous voulons en rester là. Cependant, nous avons toujours envie de l'améliorer mais rajouter des fonctions casse parfois le code, il ne marche plus. C'est dans ce but qu'existe les branches dans le suivi Git. Il s'agit à un moment de faire une empreinte, de laisser cette empreinte vierge de toute modification pour travailler sur la deuxième branche et la faire évoluer sans faire évoluer la première.

Créer une branche

git branch nom

Naviguer entre les branches

git checkout nom

Informations complémentaires

  • le nom de la première branche est master
  • pour savoir sur quelle branche nous nous trouvons git branch.
  • avant de changer de branche, il faut valider les modifications en effectuant un commit sinon la changement de branche ne se fera pas.

Fusionner deux branches

La fusion entre 2 branches s'effectue facilement avec la commande git merge nom-branche

Tout d'abord, se placer dans la branche finale habituellement master :

git checkout master

puis de fusionner avec l'autre branche test :

git merge test

Si il y a des conflits dans un fichier, corriger puis faire un commit

Supprimer une branche

git branch -d nomdelabranche
