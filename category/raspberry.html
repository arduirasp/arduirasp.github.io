Installation / Sauvegarde

Les cartes SD ont des durées de vie limitées selon le nombre de réécritures. L'installation est importante mais aussi surtout la sauvegarde de la carte sd.

Les commandes qui seront vues dans cette partie sont faites à partir d'un GNU/Linux dans un terminal. Elles sont pour la plupart à taper en administrateur.

Ecriture d'une image sur la carte sd

sudo dd if=raspbian.img of=/dev/sdb bs=4M

Cela peut être l'image de base récupéré sur le site officiel : Fondation raspberry pi ou l'image que nous avons sauvegardé (voir en dessous) sans avoir à passer par de longues configurations.

Sauvegarde de la carte SD

sudo dd if=/dev/sda | > gzip ~/image.img.gz

Sauvegarde et compression dans un fichier gz.
Pour le transférer sur une carte SD ensuite, il faudra décompresser ce fichier pour obtenir le fichier img.

Mise à jour de Raspbian

Maj des programmes

sudo apt-get update
sudo apt-get dist-upgrade

Maj du noyau

sudo rpi-update

Configuration

La configuration du Rpi est facilitée par l'utilitaire raspi-config lancé par : sudo raspi-config

Il s'agit d'étendre la partition pour utiliser toute la place, de changer le mot de passe et de changer la langue ainsi que le "timezone".

Personnalisation

Modifier le nom du Rpi

Avec l'édition des fichiers /etc/hostname et /etc/hosts

Texte d'ouverture

A l'ouverture d'une connection ssh, le texte du fichier /etc/motd s'affiche. Il peut être édité pour avoir une autre ouverture.

Note : après avoir édité toutes ces choses et fait un redémarrage, une sauvegarde peut être faite.

Notes de commandes

Forcer le hdmi

Quand le hdmi n'est pas branché au démarrage, le signal vidéo bascule sur le vga. L'édition du fichier /boot/config.txt est nécessaire :

hdmi_force_hotplug=1 et hdmi_drive=2 dans /boot/config/txt

Clé usb monté comme disque dur

Créer un répetoire avec mkdir et mount /dev/sda1 emplacement (habituellement sda1 sinon df -h pour connaître)

L'édition du fstab permet de fixer la configuration et de considérer la clé comme un disque dur (monté au démarrage) :

/dev/sda1 /home/pi/documents ext4 defaults 0 3

Le 3 définit l'ordre pour fsck (analyser la partition)
